Frequenly Asked Questions
  1. When was this built?
    This application was built in a couple of weeks in March 2010 for submission in the apps4nsw competition.
  2. What is this used for?
    Where4NSW helps you discover parts of NSW where you might want to live that you haven't considered before. It takes your price range into consideration and looks for areas that match your criteria.
  3. What technologies are used?
    jQuery is used as the javascript library. Google Project Hosting is used for hosting (using subversion) Google App Engine is used for deployment. Google Maps is used for the mapping capabilities.
  4. What license are you using?
    The code for this website is hosted here and is licensed under GPL version 3
  5. What is the need being served?
    Showcasing areas of New South Wales to more people
  6. What is the motivation behind the user interface design?
    The application is build as a webapp to allow the widest audience to use it. Nearly every computer and mobile phone has a web browser. All of the functionality is presented on one page to keep it simple. No frustrating signup is required - just use the site straight away.
  7. What data sources were used?
    • NSW state and regional indicators (housing)
  8. Does it scale?
    The functionality of the site is read-only so there should be few problems.
  9. Can I reuse some of the code?
    10. In the spirit of openness, the code is open-sourced under the GNU Public License version 3 and available here
  10. I'm thinking of a feature... could you add it to the site?
    There is plenty of room for inclusion of more search criteria -- things like what size of house you might be interested in, and general information about the areas. Since the code is open-sourced, you may want to contribute the feature yourself!
